# CivicUtility Reconciliation

The canonical catalog defines CivicUtility as a utility customer-service copilot: CSR Q&A over billing policies and service rules, resident Q&A, payment-arrangement drafting, and service-request intake integrating with Civic311.

v0.1.0 intentionally ships deterministic foundations only. It does not replace utility billing systems, process payments, operate a rate engine, decide shutoffs/reconnects, or write back to Civic311.
